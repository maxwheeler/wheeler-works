## How it works

1. A Go service polls feeds and stores raw entries in Postgres
2. A clustering step groups stories by URL canonicalization and title shingles
3. An optional summarization pass calls a local model via Ollama
4. A small web UI shows clusters, not items — one card per story, not per feed

## Why not just use Feedly

I wanted to own the data and tune the clustering. Feedly is great; this is
mine.
